Navigation Methods Comparison

Understand the difference between Declarative and Imperative navigation.

Declarative

routerLink

Best for simple template-based navigation. Handles link styling and accessibility automatically.

<a [routerLink]="['details', 101]">View</a>
Imperative

router.navigate()

Navigates based on an array of commands. Supports relative paths and dynamic segments easily.

this.router.navigate(['details', id], {{ '{' }} relativeTo: this.route {{ '}' }});
Imperative

router.navigateByUrl()

Navigates to an absolute string URL. Precise and simple for fixed paths.

this.router.navigateByUrl('/routing/details/101');

Absolute vs Relative Routing

How the path is interpreted by Angular.

Absolute

Starts from the root of the application (uses `/`).

['/routing', 'details']

Result: example.com/routing/details

Relative

Appends to the current URL active segment.

['details']

If at /routing, result: example.com/routing/details

💡 PRO TIP: In TypeScript, use {{ '{' }} relativeTo: this.route {{ '}' }} to enable relative navigation from your current component.

The "Navigation Extras" Object ⚙️

Advanced configuration for fine-tuning your navigation.

relativeTo

Required for **Relative Navigation** in TypeScript. Tells Angular which route to measure the distance from.

this.router.navigate(['details'], {{ '{' }} relativeTo: this.route {{ '}' }});

preserveQueryParams: true

Keeps existing Query Params (e.g. ?id=123) active after you navigate to a new page.

this.router.navigate(['/next'], {{ '{' }} preserveQueryParams: true {{ '}' }});

Key Insight: The Router Outlet is just a placeholder, not a container.

It doesn't wrap the new page. It just tells Angular: "Put the new page right here, next to me."

Therefore, the new page and the outlet are siblings, not parent and child!
